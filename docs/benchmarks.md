# Benchmarks

Public benchmark for esm.cpp on real OAS antibody data and synthetic batches. Results table below is regenerated by `tools/render_benchmark_table.py` from `benchmarks/results/*.json`; rows are committed alongside the code that produced them.

## Headline target

Per [SPEC.md](../SPEC.md) Phase 3 ship gate: **beats HuggingFace `EsmForMaskedLM` eager FP32 throughput** on cu_seqlens-packed batches of real OAS antibody sequences, at ESM-2-650M, on a single-socket x86 with AVX-512+VNNI. Quality drift stays inside Phase 2 bounds (PPPL < 0.1, ProteinGym Spearman < 0.01).

The dev-host row (M3 Pro NEON-Accelerate) is the floor of what's reproducible right now. The headline x86 VNNI row is the [Phase 3 retro](../notes/phase3.md) gate-machine hand-off — the harness is shipped, the recipe is in place, the production measurement is the only thing waiting on a c7i instance.

## Recommended gate machine

- **Cloud:** AWS `c7i.4xlarge` (8 physical Sapphire Rapids cores, AVX-512+VNNI+AMX, ~$0.71/h us-east-1).
- **Full socket for MKL % comparisons:** `c7i.metal-24xl`.
- **OS:** Ubuntu 22.04 LTS.
- **Toolchain:** Clang 17 or GCC 13. Pinned in `.github/workflows/ci.yml`.
- **PyTorch / HF transformers:** versions pinned in `pyproject.toml` dev extras.
- **MKL (optional, for SGEMM % gate):** Intel oneAPI Base Toolkit 2024.0+.

Disable hyperthreading on gate runs (cgroup or BIOS) so `OMP_NUM_THREADS` / `MKL_NUM_THREADS` / `ESM_NUM_THREADS` all map to physical cores.

## Reproduction commands

```bash
# 1. Build esm.cpp with bench enabled (Release).
cmake -B build -DCMAKE_BUILD_TYPE=Release -DESM_BUILD_BENCH=ON
cmake --build build -j --target bench_gemm esm_cpp_tests

# 2. Build the Python wheel against the same compiler.
pip install -e ".[dev]"

# 3. Sanity-check parity against HF on 8M / 35M.
pytest tests/python/test_against_hf.py -v

# 4. SGEMM microbenchmark vs MKL on the four critical shapes.
#    Sweep ISAs by re-running with ESM_FORCE_ISA pinned.
for isa in ref avx512 avx512vnni; do
  ESM_FORCE_ISA=$isa ./build/bench/bench_gemm \
      --benchmark_out=results-$isa.json --benchmark_repetitions=5
done

# 5. Convert + quantize the headline model (650M).
esm-cpp-convert --hf facebook/esm2_t33_650M_UR50D \
    --out weights/esm2_650m.gguf
esm-cpp-quantize --calibrate \
    --model esm2_t33_650M \
    --calib data/uniref50_calib_v1.fasta \
    --out weights/esm2_650m_calib.json
esm-cpp-quantize --apply-smoothquant \
    --model esm2_t33_650M \
    --stats weights/esm2_650m_calib.json \
    --alpha 0.5 \
    --output weights/esm2_650m_q8.gguf

# 6. Fetch the OAS sample (operator supplies the source CSV from OAS).
python tools/fetch_oas_antibodies.py \
    --input /path/to/oas_study.csv \
    --num-seqs 1000 \
    --out data/oas_sample_v1.fasta

# 7. End-to-end vs HuggingFace eager FP32 (the ship gate).
OMP_NUM_THREADS=$(nproc --all) MKL_NUM_THREADS=$(nproc --all) \
ESM_NUM_THREADS=$(nproc --all) \
esm-cpp-bench \
    --model weights/esm2_650m_q8.gguf \
    --dataset data/oas_sample_v1.fasta \
    --modes esm-cpp-int8,hf-eager-fp32 \
    --warmup 5 --iters 30 \
    --output benchmarks/results/c7i_650m_int8.json

# 8. Regenerate the table below.
python tools/render_benchmark_table.py \
    --results benchmarks/results \
    --out docs/benchmarks.md
```

The harness prints a JSON summary with esm.cpp + HF `mean_ms / p50_ms / p99_ms / throughput_seqs_per_s`, a `hardware` block (CPU model, core count, ISA, RAM), and top-level `speedup_fp32_vs_hf` / `speedup_int8_vs_hf` fields.

## Single-socket pinning

Pin the run to one socket if the machine has more than one (NUMA matters more than thread count):

```bash
numactl --cpunodebind=0 --membind=0 esm-cpp-bench --model ... \
    --dataset ... --modes esm-cpp-int8,hf-eager-fp32 \
    --output benchmarks/results/c7i_socket0_650m_int8.json
```

## Quality gate

The perf row only counts when quality holds. Run the PPPL + ProteinGym evals against the same quantized GGUF artifact:

```bash
# Pseudo-perplexity drift vs FP32.
esm-cpp-pppl --model esm2_t33_650M \
    --data data/uniref50_pppl_holdout.fasta \
    --out benchmarks/results/c7i_650m_pppl.json

# ProteinGym Spearman drift.
python -m esm_cpp.eval.proteingym \
    --model esm2_t33_650M \
    --assays-root data/proteingym_v1.3 \
    --out benchmarks/results/c7i_650m_proteingym.json
```

The Phase 2 quality gate (PPPL drift < 0.1, ProteinGym Spearman drift < 0.01) must hold at 150M and 650M for the ship row to count.
